### IMPORTANTE — Cómo funciona el deploy
- El repo está en `/apps/delfos-ai/` en el VPS (host)
- MiroFish corre dentro del container Docker `mirofish` con ruta `/app/`
- **Los archivos del host NO se montan automáticamente al container**
- Para deploy de cambios:
  1. Editar en el host (`/apps/delfos-ai/`)
  2. `docker cp` archivos individuales al container (`mirofish:/app/...`)
  3. O rebuild: `docker compose build mirofish && docker compose up -d mirofish`
  4. Después del restart: reinstalar deps con `docker exec mirofish bash -c "cd /app/backend && uv pip install fpdf2 markdown"`
  5. Fix imports: `docker exec mirofish sed -i 's/from core.agents.report_agent import/from ..services.report_agent import/' /app/backend/app/utils/pdf_generator.py`
  6. Reload backend: `docker exec mirofish bash -c "kill $(pgrep -f 'python.*run.py' | tail -1)"` (npm lo reinicia automáticamente)

### Dependencias que NO sobreviven restart del container
- `fpdf2` y `markdown` se instalaron con `uv pip install` dentro del container
- Se agregaron a `pyproject.toml` pero NO se hizo rebuild del container
- Cada vez que el container se reinicia, hay que reinstalar:
  ```bash
  docker exec mirofish bash -c "cd /app/backend && uv pip install fpdf2 markdown"
  ```
- Solución definitiva: rebuild del container (requiere fix de docker-compose.yml primero)

### Import fix que NO sobrevive restart
- `pdf_generator.py` dentro del container tiene import `from core.agents.report_agent`
- El container NO tiene `/app/core/` — ese directorio solo existe en el repo del host
- Cada restart necesita: `sed -i 's/from core.agents.report_agent import/from ..services.report_agent import/'`
- Solución definitiva: el archivo en el host (`/apps/delfos-ai/backend/app/utils/pdf_generator.py`) ya tiene el fix correcto — solo falta rebuild del container

## QUÉ SIGUE — FASE 2

### Objetivo: Restructurar el output del ReportAgent al formato de 7 secciones Delfos

El ReportAgent actual (MiroFish) genera secciones narrativas libres (2-5 secciones con títulos dinámicos). Necesitamos que genere el formato Delfos de 7 secciones fijas:

1. **Executive Summary** — Veredicto (PROCEED/PROCEED WITH CONDITIONS/DO NOT PROCEED), confidence score, 3 párrafos
2. **Critical Variables** — 3-5 variables clave con valor actual, rango realista, sensibilidad
3. **Key Risks** — Máximo 5 riesgos con probabilidad, impacto, mitigación
4. **Structural Weaknesses** — Debilidades sistémicas del modelo
5. **Scenario Outcomes** — 3-5 escenarios con probabilidades y proyecciones 12/24 meses
6. **Decision Impact** — Si procede vs si no procede
7. **Recommended Actions** — Acciones priorizadas con deadlines

### Approach técnico:
- NO modificar el ReportAgent core (es complejo, 2578 líneas, ReACT mode)
- Crear un **post-processor** que tome el markdown del reporte + el Devil's Advocate y los re-estructure en las 7 secciones usando DeepSeek API
- O: modificar los prompts del ReportAgent (`PLAN_SYSTEM_PROMPT` y `SECTION_SYSTEM_PROMPT_TEMPLATE` en `core/agents/report_agent.py`) para que genere las 7 secciones directamente
